# Project Goals

## Mission
Build a next‑generation 3D printing preparation environment: a custom slicer with integrated pre‑support and file‑preparation tools that helps users understand, prepare, and validate models quickly and confidently. The focus is decision support and reliable outcomes—not just rendering—prioritizing clarity, speed, accuracy, and an architecture that can expand into adjacent domains (e.g., bracing, measurement, export).

This document provides high‑level intent and scope to align contributors. It deliberately avoids implementation details and specific file references.

## Vision
- Build a custom, modern 3D printing slicer coupled with a practical suite of pre‑support and file‑preparation tools.
- Prioritize clarity, speed, and decision‑making for preparing models for successful prints.

## Tech Stack (Non‑Binding)
- **Framework and language**: TypeScript with a modern React/Next.js setup.
- **3D**: three.js with a React renderer (e.g., react‑three‑fiber) and common helpers (e.g., drei).
- **Styling**: Utility‑first CSS (e.g., Tailwind) with a small layer of custom styles.

## Scope
- **Custom slicer core**
  - Foundation for layer planning, print analysis, and future toolchains.
- **Pre‑support tools**
  - Utilities to understand and mitigate unsupported regions and related risks.
- **File preparation tools**
  - Orientation, cleanup, and basic prep to get models ready for reliable printing.
- **Extensible domains**
  - Structured to add areas like bracing, measurement, and export later without major rewrites.

## Product Pillars
- **Usability**
  - Minimal, discoverable controls and a cohesive UX.
- **Accuracy**
  - Predictable coordinate conventions and consistent units.
- **Performance**
  - Interactive feedback even as model complexity grows.
- **Extensibility**
  - Modular architecture that accommodates new domains cleanly.

## Non‑Goals (Initial Phase)
- Full G‑code generation or printer‑specific workflows.
- Complex mesh repair or CAD‑level editing.
- Multi‑object layout optimization beyond basic utilities.

## Success Criteria
- Users can prepare models for printing with confidence and minimal friction.
- Pre‑support insights align with user intuition for “unsupported” 3D volumes.
- The interface remains responsive and understandable without documentation.

## Tech Direction (Non‑Binding)
- **Framework and language**: TypeScript with a modern React/Next.js setup.
- **3D**: three.js with a React renderer (e.g., react‑three‑fiber) and common helpers (e.g., drei).
- **Styling**: Utility‑first CSS (e.g., Tailwind) with a small layer of custom styles.
